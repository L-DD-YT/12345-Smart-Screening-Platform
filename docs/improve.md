# 关于公益诉讼检察对接12345民生热线数据的筛查需求建议

## 1. 存在的困难

### **1.1 依靠人工逐条查看，筛查效率低下**
当前工作模式为：定期对七部推送的工单列表进行简单筛查，逐条点开阅读详情判断。面对海量数据，人工筛查犹如 **“大海捞针”** ，极易因视觉疲劳和时间不足导致线索遗漏，无法对全区民生数据进行 **全覆盖、无死角的深度排查**。

### **1.2 重复反映、同源投诉数据充斥，严重消耗办案精力**
12345平台为客观记录每一次群众来电，对同一事项（如同一处井盖缺失、同一工厂夜间排污）的多次来电均生成为独立工单。目前 **缺乏技术手段** 对重复、衍生投诉进行 **聚合去重**。检察官在人工查看时，经常遇到以下情况：
- 对于某小区垃圾堆积问题，连续多日出现数十条不同居民就同一事件的投诉工单；
- 对于某施工扰民问题，不同投诉人使用的地址描述略有差异，人工难以快速识别为同一事源。

这些重复冗余数据导致检察官 **大量精力耗费在“识别是否为新线索”而非“研判是否成案”** ，严重挤占了实质性的调查研判时间。

### **1.3 公益与私益界限模糊，人工甄别难度大、标准不统一**
12345平台以解决群众个人诉求为首要目标，工单内容多为“我家楼下……”“我个人遇到……”等具体 **私益表达**。检察官需逐条分析工单中隐含的 **公益属性**。例如：
- 反映“我家水龙头出水浑浊” → 背后可能是涉及不特定多数人的 **小区供水管网污染**；
- “个人医保报销难” → 也可能折射出 **医保基金管理漏洞**。

但在人工逐条阅读的有限时间里，检察官难以对每条工单的公益属性进行深度挖掘和准确归类，**判断标准因人而异，缺乏统一、智能的甄别辅助工具**。

### **1.4 线索研判缺乏历史数据比对，难以评估“屡诉未决”的严重程度**
判断一条线索是否属于行政机关履职不到位、是否需要检察公益诉讼介入，关键要看该问题是否为 **“长期存在、多次反映、仍未解决”** 。当前：
- 仅能看到单条工单的当次处理结果；
- **无法快速调取**同一地点、同类问题的历史投诉记录和办结满意度评价。

检察官若想了解某一污染源过去半年的投诉走势，需 **手动变换关键词和日期反复检索，再自行汇总统计**，操作极为繁琐，导致对问题严重性、紧迫性的评估依据不足，难以及时启动公益诉讼监督程序。

### **1.5 缺乏可视化与趋势分析手段，难以发现区域性、系统性治理漏洞**
人工筛查仅能处理 **“点状”信息**，无法将散落在不同街镇、不同时段的投诉工单进行 **空间聚合与趋势预判**。例如：
- 某类预付卡消费纠纷是否在全区范围内出现蔓延苗头；
- 某流域上下游是否存在连续污染投诉。

这些都需要借助 **数据分析工具** 才能直观呈现，难以从宏观层面发现行业性、区域性的公益损害风险。

## 2. 意见建议

### **（一）公益损害线索智能识别与分类筛选**

> **需求描述**：依托公益诉讼法定领域框架，建立多维度关键词库和语义识别模型，实现12345热线工单中公益损害线索的自动识别、分类标注与优先级排序。

**核心要求：**

1. **领域分类覆盖**  
   按现行 **“4+11”法定公益诉讼领域** 进行分类标注，涵盖：
   - 生态环境和资源保护
   - 食品药品安全
   - 安全生产
   - 国有财产保护
   - 国有土地使用权出让
   - 未成年人保护
   - 个人信息保护
   - 妇女权益保障
   - 无障碍环境建设
   - 文物保护  
   同时 **预留新增领域扩展接口**，以适应公益诉讼法定范围的不断扩大。

2. **公益与私益区分**  
   建立智能过滤规则，精准识别“公益损害”与“私益损害”。  
   **核心区分标准**：
   - 若涉案利益仅与特定个人存在直接利害关系 → **私益**
   - 若被损害利益涉及 **不特定多数人或者国家利益** → **公益**  
   需结合工单中投诉主体数量、利益波及范围、涉及人群等要素，通过 **语义分析自动判断公益属性**。

3. **关键词匹配 + 语义理解**  
   进一步扩展和优化关键词库，对工单内容进行深度解析，避免仅依赖关键词匹配导致的线索遗漏或误判。系统应实现对生态环境和资源保护、食品药品安全、安全生产、特殊群体权益保护等重点领域的 **高精度识别**。

4. **投诉频次与优先级分级**  
   系统应自动统计同一事项或同一地理位置的投诉频次，按投诉频次自动划分线索优先级（**高、中、低**），帮助检察官快速锁定重点民生诉求。  
   - 对 **高频集中投诉、群体性投诉** 自动标记为 **高优先级线索**。

### **（二）重复反映与未解决诉求预警机制**

> **需求描述**：对同一事项多次反映、群众不满意、行政机关处理不到位等情形进行智能化监测、预警和分类，为公益诉讼检察监督介入提供精准靶向。

**核心要求：**

1. **重复投诉识别与聚合**  
   对同一事项或相同衍生事项的多次致电工单进行 **智能聚合**，自动识别重复投诉。聚合维度包括：
   - 投诉人信息
   - 投诉地址/地理位置
   - 问题描述相似度
   - 涉及主体

2. **“多次反映未解决”分级预警**  
   - 按反映次数（如 **3次以上、5次以上、10次以上**）和持续时长（如 **超过30天未解决**）设定分级预警阈值，自动生成预警提示。
   - 自动识别 **“办理结果不满意”“整改未达预期”** 等情形，重点标记行政机关已多次处理但群众仍不满意的 **“疑难工单”**。

3. **行政机关履职异常监测**  
   重点在 **工单总数、解决率、满意率、响应率** 等指标维度进行分析研判，对某一区域或领域出现 **解决率持续偏低、满意率骤降** 等异常情况的，自动预警并提示关注该区域或领域行政机关履职情况。

4. **趋势分析**  
   对某一类型公益损害问题的投诉数量变化趋势进行统计分析，**及时发现新兴问题和阶段性高发问题**，为 **预防性公益诉讼** 提供数据支撑。

### **（三）线索自动推送与智能辅助研判**

> **需求描述**：建立 **“T+1”或实时同步机制**，实现高质量线索的自动推送，并提供智能辅助研判功能。

**核心要求：**

1. **定时批量推送**  
   系统 **按日或按周** 将涉及法定公益诉讼领域的投诉举报信息批量推送至公益诉讼部门，实现与12345平台投诉内容的 **实时同步**。

2. **智能研判辅助**  
   参照 **“线索—法条—案例”智能推送系统** 建设思路，整合公益诉讼相关法律法规和典型案例库，实现：
   - 线索与 **违法情形认定标准**、**监管职责法条** 的精准匹配；
   - 辅助检察官快速研判线索 **成案可能性**；
   - 根据线索具体情况，结合所涉及领域特点，合理评估 **社会影响、涉案范围、紧急程度** 等要素，并综合线索真实性与调查取证难度，进行 **分类定级**。

3. **专项分析报告自动生成**  
   对 **高频集中诉求、重点诉求人、重要问题** 等线索，自动生成专项分析报告，按 **月度或季度** 定期推送至公益诉讼部门，为专项监督行动提供数据支撑。

4. **紧急线索即时推送**  
   对涉及 **重大公共安全风险**、可能造成 **严重公益损害** 的紧急线索，实现 **即时识别和即时推送**，确保第一时间介入处置。
